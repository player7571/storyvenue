# StoryVenue

이 문서는
`이미 프로젝트를 clone 받았고`
`server/.env 파일도 전달받은 팀원`
기준으로 적었습니다.

즉, 이 문서에서 할 일은 딱 두 가지입니다.

1. Docker 로 서버 켜기
2. Android Studio 로 앱 실행하기

Supabase 설정은 이미 끝나 있다고 가정합니다.

## 1. 준비물

아래 두 개만 있으면 됩니다.

- `Docker Desktop`
- `Android Studio`

## 2. 폴더 확인

프로젝트 폴더 안에 아래가 있어야 합니다.

- `docker-compose.yml`
- `server/.env`
- `app/`

만약 `server/.env` 가 없으면
프로젝트 담당자에게 다시 받아야 합니다.

## 3. 서버 켜기

### 3-1. Docker Desktop 실행

먼저 `Docker Desktop` 을 켭니다.

중요:
- 그냥 앱만 열지 말고
- Docker 가 완전히 켜질 때까지 기다리세요

보통 상단이나 창 안에서
`Engine running`
비슷한 표시가 보이면 됩니다.

### 3-2. 터미널 열기

프로젝트 폴더에서 터미널을 엽니다.

즉, 지금 위치가 이런 상태여야 합니다.

```bash
cd storyvenue
```

### 3-3. 서버 실행 명령 입력

아래 명령을 그대로 복사해서 넣으세요.

```bash
docker compose up -d --build
```

처음에는 조금 오래 걸릴 수 있습니다.  
정상입니다.

### 3-4. 서버가 켜졌는지 확인

아래 명령을 입력하세요.

```bash
curl http://127.0.0.1:8000/health
```

이렇게 나오면 성공입니다.

```json
{"status":"ok"}
```

## 4. Android Studio 로 앱 실행하기

### 4-1. Android Studio 열기

`Android Studio` 를 엽니다.

### 4-2. 프로젝트 열기

`app/` 폴더를 엽니다.

즉,
`storyvenue/app`
폴더를 열면 됩니다.

### 4-3. 실행할 앱 확인

상단의 실행 대상이 `app` 인지 확인하세요.

## 5. 에뮬레이터로 실행하기

가장 쉬운 방법입니다.

### 5-1. 에뮬레이터 만들기

Android Studio 에서:

1. `Device Manager` 열기
2. `Create device` 누르기
3. 아무 Pixel 기기 하나 선택
4. Android 버전 하나 선택
5. `Finish`

### 5-2. 에뮬레이터 켜기

만든 기기 옆의 실행 버튼을 누릅니다.

에뮬레이터가 완전히 켜질 때까지 기다립니다.

### 5-3. 앱 실행

Android Studio 상단의 `Run` 버튼을 누릅니다.

그러면 앱이 에뮬레이터에 설치되고 실행됩니다.

### 5-4. 서버 주소 확인

에뮬레이터에서는 서버 주소가 아래여야 합니다.

```text
http://10.0.2.2:8000
```

이 주소는
`에뮬레이터에서 내 컴퓨터 서버로 접속하는 주소`
입니다.

앱 첫 화면이나 설정 입력칸에 서버 주소가 보이면
이 값으로 되어 있는지 확인하세요.

## 6. 진짜 안드로이드 휴대폰으로 실행하기

에뮬레이터 대신
직접 휴대폰으로 실행할 수도 있습니다.

### 6-1. 휴대폰과 컴퓨터를 같은 와이파이에 연결

이게 가장 중요합니다.

둘이 같은 와이파이에 연결되어 있어야 합니다.

### 6-2. 휴대폰에서 개발자 옵션 켜기

휴대폰에서:

1. `설정`
2. `휴대전화 정보`
3. `빌드 번호` 여러 번 누르기
4. 개발자 옵션 활성화

### 6-3. USB 디버깅 켜기

휴대폰에서:

1. `설정`
2. `개발자 옵션`
3. `USB 디버깅` 켜기

### 6-4. USB 로 컴퓨터와 연결

휴대폰을 컴퓨터와 USB 로 연결합니다.

휴대폰 화면에
`이 컴퓨터를 허용할까요?`
같은 창이 뜨면 허용하세요.

### 6-5. Android Studio 에서 휴대폰 선택

상단 실행 기기 목록에서
연결된 휴대폰을 선택합니다.

### 6-6. 앱 실행

`Run` 버튼을 누릅니다.

### 6-7. 휴대폰에서는 서버 주소를 바꿔야 함

휴대폰에서는 `10.0.2.2` 를 쓰면 안 됩니다.

휴대폰에서는
`내 컴퓨터의 로컬 IP 주소`
를 써야 합니다.

예:

```text
http://192.168.0.15:8000
```

## 7. 내 컴퓨터 IP 주소 찾기

### Mac

터미널에서 아래 둘 중 하나를 입력하세요.

```bash
ipconfig getifaddr en0
```

또는

```bash
ipconfig getifaddr en1
```

숫자가 나오면
그걸 앱 서버 주소에 넣으면 됩니다.

예:

```text
http://192.168.0.15:8000
```

### Windows

명령 프롬프트에서:

```bash
ipconfig
```

그리고 `IPv4 Address` 값을 찾으면 됩니다.

## 8. 실행이 잘 됐는지 확인하는 순서

앱이 켜졌다면 아래 순서대로 해보세요.

1. 회원가입
2. 로그인
3. 새 인터뷰 세션 만들기
4. 음성 인터뷰하기
5. 장 초안 만들기
6. 책 저장하기
7. 피드에 올리기

여기까지 되면
기본 실행은 성공입니다.

## 9. 자주 쓰는 Docker 명령

### 서버 켜기

```bash
docker compose up -d --build
```

### 서버 상태 보기

```bash
docker compose ps
```

### 서버 로그 보기

```bash
docker compose logs -f storyvenue-api
```

### 서버 끄기

```bash
docker compose down
```

## 10. 자주 생기는 문제

### 1. `curl http://127.0.0.1:8000/health` 가 안 됨

확인할 것:

- Docker Desktop 이 진짜 켜져 있는지
- `docker compose up -d --build` 를 실행했는지
- `docker compose ps` 했을 때 `storyvenue-api` 가 떠 있는지

### 2. Android Studio 에서 실행 버튼이 안 됨

확인할 것:

- `app` 프로젝트를 연 게 맞는지
- 에뮬레이터나 휴대폰이 연결되어 있는지

### 3. 에뮬레이터에서는 되는데 휴대폰에서는 안 됨

확인할 것:

- 휴대폰과 컴퓨터가 같은 와이파이인지
- 앱 서버 주소가 `http://내_컴퓨터_IP:8000` 인지
- `10.0.2.2` 를 그대로 쓰고 있지 않은지

### 4. 음성이 안 됨

확인할 것:

- 휴대폰/에뮬레이터에서 마이크 권한 허용했는지
- 서버 로그에 에러가 있는지

로그 확인:

```bash
docker compose logs -f storyvenue-api
```

## 11. 제일 짧은 버전

정말 짧게 하면 이렇습니다.

1. Docker Desktop 켜기
2. 프로젝트 폴더에서 `docker compose up -d --build`
3. `curl http://127.0.0.1:8000/health`
4. Android Studio 로 `storyvenue/app` 열기
5. 에뮬레이터 켜기
6. `Run`
7. 서버 주소가 `http://10.0.2.2:8000` 인지 확인

실기기라면 마지막 주소만 이렇게 바꾸면 됩니다.

```text
http://내_컴퓨터_IP:8000
```
