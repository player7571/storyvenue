# Prompt.md

## 서비스 한 줄 설명
노인 사용자가 음성으로 AI와 대화하며 자신의 삶을 기록하고,
그 대화 내용을 기반으로 장별 자서전 초안을 생성하는 안드로이드 앱

## 문제 정의
많은 노인 사용자는 긴 글을 직접 입력하는 데 부담을 느낀다.
무엇부터 써야 할지 모르거나,
기억을 길게 정리하는 과정 자체가 어렵고 피로할 수 있다.
텍스트 입력보다 말하기가 더 편한 사용자도 많다.

## 해결 방식
앱은 음성 인터뷰형 챗봇을 제공한다.
AI가 쉬운 질문을 하나씩 말하면 사용자는 음성으로 답한다.
시스템은 음성을 텍스트로 변환해 저장하고,
핵심 기억을 구조화한 뒤,
장별 자서전 초안으로 재구성한다.

## 핵심 사용자
- 60대 이상 고령 사용자
- 스마트폰 사용이 익숙하지 않을 수 있는 사용자
- 긴 글쓰기보다 말하기가 더 편한 사용자
- 자신의 삶을 기록하고 싶은 사용자

## 핵심 인터랙션
- 기본 입력: 음성
- 기본 출력: 음성 + 화면 텍스트
- 모든 발화는 텍스트 로그로 저장
- 자서전 생성은 저장된 텍스트 로그와 memory item을 기반으로 수행
- 사용자는 필요 시 텍스트 화면으로도 확인할 수 있다

## MVP 기능
1. 이메일 회원가입 / 로그인
2. 인터뷰 세션 생성
3. 마이크 버튼으로 음성 입력
4. 음성 업로드 및 STT
5. 인식 결과를 텍스트 로그로 저장
6. AI가 다음 질문 또는 짧은 반응 생성
7. TTS 음성 재생
8. 답변 기반 memory item 추출
9. 장별 자서전 초안 생성
10. 초안 수정 / 재생성
11. 전체 자서전 버전 저장

## MVP에서 제외
- 전화망 기반 자동 통화
- 완전 실시간 speech-to-speech
- 가족 공동 편집
- PDF 다운로드
- 푸시 알림
- 감정 점수화
- 고도화된 벡터 검색
- 관리자 페이지

## 핵심 UX 원칙
1. 한 번에 질문 하나만 한다.
2. 짧고 쉬운 문장으로 말한다.
3. 중요한 정보는 확인한다.
4. 사용자가 답하지 못하면 힌트를 준다.
5. 음성만이 아니라 텍스트도 항상 함께 보여준다.
6. 인터뷰가 심문처럼 느껴지지 않도록 공감적이고 차분한 말투를 유지한다.
7. 사용자가 잘 못 들었을 때 다시 듣기 경로를 제공한다.
8. 사용자가 오래 생각해도 재촉하지 않는다.

## 핵심 흐름
1. 사용자가 로그인한다.
2. 새 인터뷰 세션을 시작한다.
3. AI가 짧은 질문을 음성과 텍스트로 제시한다.
4. 사용자가 버튼을 눌러 음성으로 답한다.
5. 서버가 STT를 수행하고 사용자 발화를 텍스트로 저장한다.
6. 서버가 memory item 을 추출한다.
7. 서버가 다음 질문 또는 짧은 반응을 생성한다.
8. 응답을 텍스트와 TTS로 제공한다.
9. memory item 이 쌓이면 장별 초안을 생성한다.
10. 사용자는 초안을 수정하거나 다시 생성한다.
11. 최종 자서전 버전을 저장한다.

## 예시 memory item
사용자 답변:
"초등학교 때 여름마다 할머니 댁에 갔는데 그때가 가장 좋았어요."

추출 결과 예시:
- period: 초등학교 시절
- place: 할머니 댁
- person: 할머니
- emotion: 행복, 그리움
- event: 여름방학 방문
- meaning: 어린 시절 가장 따뜻한 기억

## chapter 예시
- 1장 어린 시절
- 2장 가족과 집
- 3장 학교와 성장
- 4장 힘들었던 시기
- 5장 지금의 나

## 기술 스택
- Android Kotlin
- FastAPI
- Supabase
- OpenAI API

## 보안 원칙
- OpenAI API key 는 앱에 넣지 않는다.
- OpenAI 호출은 서버에서만 수행한다.
- 사용자 데이터는 인증된 사용자 기준으로 접근 제어한다.
- 음성 파일은 최소 보관 원칙을 따른다.
- 텍스트 로그와 memory item 은 사용자 본인만 접근 가능해야 한다.
