# SAFETY.md

## Priority
안전 관련 발화는 자서전 인터뷰보다 우선한다.

## Escalation triggers
다음과 같은 표현은 안전 우선 대응 대상으로 본다.
- 숨이 차다
- 가슴이 아프다
- 쓰러졌다
- 일어날 수 없다
- 너무 아프다
- 죽고 싶다
- 사라지고 싶다
- 사는 의미가 없다
- 무섭다 / 큰일 났다 / 도와달라
- 집에 혼자 있는데 위험하다

## Agent behavior
- 일반 인터뷰 흐름을 즉시 중단한다.
- 짧고 분명한 안전 확인 문장으로 전환한다.
- 응급 가능성이 있으면 주변 사람 도움 또는 응급 도움 요청을 우선 안내한다.
- 진단하지 않는다.
- 자서전 질문을 계속하지 않는다.
- 필요 시 보호자나 주변 사람에게 도움을 요청하도록 안내한다.

## Example safe responses
- "지금은 인터뷰보다 안전이 더 중요합니다."
- "주변에 도움을 요청할 수 있는 분이 계신가요?"
- "응급상황처럼 느껴지면 바로 119에 도움을 요청해 주세요."
- "혼자 계시면 가까운 분께 바로 연락해 보세요."

## Failure policy target
- 2~3회 연속 음성 인식 실패 시 텍스트 확인 또는 다시 말하기를 유도한다.
- 잘못 들은 정보는 저장 전 확인한다.
- 의미가 불분명한 위험 표현은 단정하지 말고 짧게 다시 확인한다.

## Logging policy
- safety_mode 진입 여부를 메시지 단위로 기록한다.
- 위험 발화 원문은 최소한으로 저장하고, 권한 제어를 강화한다.

## Current minimum implementation
- 현재 server 는 `/voice/turn` 과 `/safety/check` 에서 위험 발화를 검사한다.
- 고위험으로 판단되면 일반 인터뷰 질문 대신 짧은 안전 안내 응답으로 전환한다.
- user / assistant 메시지에는 `safety_mode` 가 기록된다.
- 연속 인식 실패가 쌓이면 앱에서 더 천천히 다시 말하도록 복구 안내를 표시한다.
- 더 넓은 위험 표현, 완곡 표현, 마스킹 정책은 실제 사용 로그 기반 추가 튜닝이 필요하다.
