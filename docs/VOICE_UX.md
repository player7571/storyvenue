# VOICE_UX.md

## Target user
- 60대 이상 고령 사용자
- 스마트폰 사용이 익숙하지 않을 수 있음
- 긴 텍스트 입력보다 말하기가 더 편할 수 있음
- 청력, 반응속도, 인지부하를 고려해야 함

## UX principles
- 질문은 짧고 단순하게
- 한 번에 하나의 과업만 제시
- 상태를 항상 알려주기
  - 듣는 중
  - 변환 중
  - 답변하는 중
  - 재생 중
- 다시 말하기 버튼 제공
- 마지막 질문 다시 듣기 기능 제공
- 마지막 인식 결과를 텍스트로 보여주기
- "맞나요?" 확인 패턴 사용

## Recommended interaction model
- 기본은 반이중 push-to-talk
- 마이크 버튼을 누르고 말한다
- 사용자가 말하는 동안 앱은 듣기만 한다
- 사용자가 말을 마치면 변환 후 답변한다
- 모든 음성 턴은 텍스트로도 보인다

## Screen requirements
### Voice Interview Screen
필수 요소:
- 큰 마이크 버튼
- 현재 상태 텍스트
- 마지막 assistant 질문 텍스트
- 사용자 인식 결과 텍스트
- 다시 듣기 버튼
- 다시 말하기 버튼
- 세션 종료 버튼

### Draft Screen
필수 요소:
- 장 제목
- 본문 텍스트
- 수정 요청 버튼
- 다시 생성 버튼
- 저장 상태 표시

## Copy style
- 짧고 쉬운 한국어
- 존댓말
- 과장 없는 따뜻한 표현
- 한 문장에 정보 과다 금지

## Things to avoid
- 너무 빠른 말투
- 너무 긴 질문
- 한 턴에 여러 질문
- 확인 없이 민감한 정보 저장
- 작은 버튼과 복잡한 화면
- 음성만 제공하고 텍스트를 숨기는 방식

## Recovery patterns
- 인식 실패 시:
  - "제가 잘 못 들었어요. 한 번만 더 천천히 말씀해 주세요."
- 사용자가 답하기 어려워할 때:
  - "괜찮으시면 장소 이름만 먼저 말씀해 주세요."
- 사용자가 잘 못 들었을 때:
  - "제가 다시 말씀드릴게요."
